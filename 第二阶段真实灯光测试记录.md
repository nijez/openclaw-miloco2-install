## 验收结论

本次测试不能按 API `success:false` 直接判定失败。现场灯光已经发生变化，说明“小龙虾 → 馨光 Skill → wainfort-server → 馨光设备”的业务链路已经打通。

后续需要继续排查 wainfort-server 为什么实际控制成功但返回 `success:false`。在二进制修复前，第二阶段验收口径先按现场反馈分层判断：

1. 请求是否已发送。
2. 目标设备是否存在。
3. 用户现场是否看到灯光变化。
4. API 返回状态只作为参考，不单独作为 RGBCW 灯测试失败依据。
5. 如果出现连续变色、关灯再亮或自动切换颜色，应记录为 `UNSTABLE_MULTIPLE_COMMANDS`，本轮不能作为稳定验收，下一轮必须只发送一次请求并等待现场确认。
