1. 只控制指定设备，不要自动控制其它设备。
2. 每次测试只发送一次灯光请求。
3. 请求发送后立即停止，等待我观察实物灯光。
4. 只通过馨光 Skill / wainfort-server API 发送灯光请求。
5. 不要用普通 miloco 直接控制结果反向判定 RGBCW 灯失败。
6. 不要仅根据 API success 字段判断成败。
7. 不要自动重试、关灯、开灯、切换颜色继续验证。
8. 不要控制其它设备做对照。

指定设备：

```text
家庭：林坞店
房间：二楼客厅
设备：RGBCW灯柱
效果：马尔代夫灯光效果
```

请按下面流程执行：

1. 确认目标设备是「林坞店 / 二楼客厅 / RGBCW灯柱」。
2. 进入单次测试模式，记录 `LIGHT_TEST_SINGLE_SHOT`。
3. 调用馨光 Skill 发送一次「马尔代夫灯光效果」请求。
4. 记录 `LIGHT_REQUEST_SENT`。
5. 如果 API 返回 `success:false`，记录 `LIGHT_API_RETURNED_FALSE`，但不要直接判定失败，也不要继续控制设备。
6. 回复：

```text
灯光请求已发送，请观察目标设备是否发生变化。

如果没有变化，请回复“未变化”。
```

如果我回复「已变化」：

1. 记录 `PHYSICAL_CHANGED`。
2. 记录 `LIGHT_TEST_SUCCESS`。
3. 如果之前 API 返回 `success:false`，同时记录 `PHYSICAL_SUCCESS_API_FALSE`。

如果我回复「未变化」：

1. 记录 `PHYSICAL_NOT_CHANGED`。
2. 记录 `LIGHT_TEST_FAILED`。
3. 汇报：现场未观察到灯光变化，需要继续排查。

如果我反馈灯光连续变化、自动关灯、自动变成其它颜色，或多次变化：

1. 记录 `UNSTABLE_MULTIPLE_COMMANDS`。
2. 汇报：控制链路已触达设备，但本轮出现多条控制命令叠加，不能作为稳定验收。
3. 下一轮必须只发送一次请求并等待现场确认。
