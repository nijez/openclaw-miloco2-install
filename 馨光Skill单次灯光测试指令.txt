请执行「馨光 Skill 单次灯光测试」。全程必须使用中文回复。

测试目标：

```text
家庭：林坞店
房间：二楼客厅
设备：RGBCW灯柱
测试效果：森林晨光
```

重要规则：

1. 只控制上面指定的设备。
2. 只发送一次灯光请求。
3. 请求发送后立即停止，等待我观察现场灯光。
4. 不要自动重试。
5. 不要自动关灯。
6. 不要自动开灯。
7. 不要切换其它颜色继续验证。
8. 不要控制其它设备做对照。
9. 不要调用 `miloco-cli device control` 直接控制 RGBCW 设备。
10. 不要仅根据 API `success` 字段判断成败。

允许执行的只读检查：

```text
/api/status
/api/devices
设备列表查询
Skill 是否安装
服务是否运行
```

请按下面流程执行：

1. 确认目标设备是「林坞店 / 二楼客厅 / RGBCW灯柱」。
2. 进入单次测试模式，记录 `LIGHT_TEST_SINGLE_SHOT`。
3. 只通过馨光 Skill 发送一次「森林晨光」灯光请求。
4. 请求发送后记录 `LIGHT_REQUEST_SENT`。
5. 如果 API 返回 `success:false`，记录 `LIGHT_API_RETURNED_FALSE`，但不要继续控制设备。
6. 回复下面这段话后停止：

```text
灯光请求已发送，请观察目标设备是否发生变化。
如果灯光已变化，请回复“已变化”。
如果灯光未变化，请回复“未变化”。
```

如果我回复「已变化」：

1. 记录 `PHYSICAL_CHANGED`。
2. 记录 `LIGHT_TEST_SUCCESS`。
3. 如果之前 API 返回 `success:false`，同时记录 `PHYSICAL_SUCCESS_API_FALSE`。
4. 汇报：实际控制成功，API 返回状态需修复。

如果我回复「未变化」：

1. 记录 `PHYSICAL_NOT_CHANGED`。
2. 记录 `LIGHT_TEST_FAILED`。
3. 汇报：现场未观察到灯光变化，需要继续排查。

如果我反馈灯光连续变化、自动关灯、自动变成其它颜色，或多次变化：

1. 记录 `UNSTABLE_MULTIPLE_COMMANDS`。
2. 汇报：控制链路已触达设备，但本轮出现多条控制命令叠加，不能作为稳定验收。
3. 下一轮必须只发送一次请求并等待现场确认。
